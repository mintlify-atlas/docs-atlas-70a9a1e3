This configuration includes multiple colorscheme options with **onedark** set as the default theme.

## Configuration File

Colorschemes are defined in `lua/plugins/colorscheme.lua`.

```lua
return {
  { "rebelot/kanagawa.nvim" },
  { "NLKNguyen/papercolor-theme" },
  { "ellisonleao/gruvbox.nvim" },
  { "rose-pine/neovim" },
  { "tiagovla/tokyodark.nvim" },
  { "sainnhe/everforest" },
  {
    "navarasu/onedark.nvim",
    opts = {
      highlights = {
        Visual = { bg = "#ffff00" },
      },
    },
  },
  {
    "LazyVim/LazyVim",
    opts = {
      colorscheme = "onedark",
    },
  },
}
```

Location: `lua/plugins/colorscheme.lua:1-23`

## Changing Colorschemes

### Method 1: Runtime Switch (Temporary)

Switch colorschemes during your session:

```vim
:colorscheme kanagawa
:colorscheme gruvbox
:colorscheme rose-pine
:colorscheme everforest
:colorscheme tokyodark
```

This change is temporary and will reset when you restart Neovim.

### Method 2: Persistent Configuration

Edit `lua/plugins/colorscheme.lua` and change the default:

<Accordion title="Example: Switch to Kanagawa">
```lua
return {
  { "rebelot/kanagawa.nvim" },
  { "NLKNguyen/papercolor-theme" },
  { "ellisonleao/gruvbox.nvim" },
  { "rose-pine/neovim" },
  { "tiagovla/tokyodark.nvim" },
  { "sainnhe/everforest" },
  {
    "navarasu/onedark.nvim",
    opts = {
      highlights = {
        Visual = { bg = "#ffff00" },
      },
    },
  },
  {
    "LazyVim/LazyVim",
    opts = {
      colorscheme = "kanagawa",  -- Changed from "onedark"
    },
  },
}
```
</Accordion>

### Method 3: LazyVim Picker

Use the built-in colorscheme picker:

```vim
<leader>uC
```

Or via command:

```vim
:Telescope colorscheme
```

## Customizing Colorschemes

### Modifying Highlight Groups

You can customize any installed colorscheme by modifying its `opts` table:

```lua
{
  "navarasu/onedark.nvim",
  opts = {
    style = "dark", -- dark, darker, cool, deep, warm, warmer
    transparent = false,
    highlights = {
      Visual = { bg = "#ffff00" },
      CursorLine = { bg = "#2d3139" },
      LineNr = { fg = "#4b5263" },
    },
  },
}
```

### Example: Transparent Background

<Accordion title="Make Background Transparent">
```lua
{
  "navarasu/onedark.nvim",
  opts = {
    transparent = true,
    highlights = {
      Visual = { bg = "#ffff00" },
    },
  },
}
```
</Accordion>

### Example: Customize Rose Pine

<Accordion title="Rose Pine Customization">
```lua
{
  "rose-pine/neovim",
  name = "rose-pine",
  opts = {
    variant = "moon", -- auto, main, moon, or dawn
    dark_variant = "moon",
    disable_italics = false,
    highlight_groups = {
      Visual = { bg = "surface" },
    },
  },
}
```
</Accordion>

## LazyVim Colorscheme Integration

### Install Fallback

Lazy.nvim is configured with fallback colorschemes during installation:

```lua
install = { colorscheme = { "tokyonight", "habamax" } }
```

Location: `lua/config/lazy.lua:33`

This ensures a colorscheme is available even if plugins fail to install.

### Setting the Active Theme

The active colorscheme is set via LazyVim configuration:

```lua
{
  "LazyVim/LazyVim",
  opts = {
    colorscheme = "onedark",
  },
}
```

Location: `lua/plugins/colorscheme.lua:18-22`

## Adding New Colorschemes

To add a new colorscheme:

1. Add the plugin to `lua/plugins/colorscheme.lua`:

```lua
return {
  -- ... existing themes ...
  { "catppuccin/nvim", name = "catppuccin" },
  
  {
    "LazyVim/LazyVim",
    opts = {
      colorscheme = "catppuccin",  -- Set as default
    },
  },
}
```

2. Restart Neovim or run:

```vim
:Lazy sync
```

3. Switch to the new theme:

```vim
:colorscheme catppuccin
```

## Troubleshooting

<Accordion title="Colors look wrong in terminal">
  ### Terminal Color Support
  
  Ensure your terminal supports 24-bit true color:
  
  ```lua
  vim.opt.termguicolors = true
  ```
  
  This is enabled by default in LazyVim.
  
  Test true color support:
  ```bash
  curl -s https://gist.githubusercontent.com/lifepillar/09a44b8cf0f9397465614e622979107f/raw/24-bit-color.sh | bash
  ```
</Accordion>

<Accordion title="Colorscheme not loading">
  ### Plugin Not Installed
  
  Check if the colorscheme plugin is installed:
  
  ```vim
  :Lazy
  ```
  
  Look for the colorscheme plugin and ensure it's loaded (green checkmark).
  
  Force reinstall:
  ```vim
  :Lazy clean
  :Lazy sync
  ```
</Accordion>

<Accordion title="Custom highlights not applying">
  ### Load Order Issue
  
  Ensure your colorscheme configuration has priority:
  
  ```lua
  {
    "navarasu/onedark.nvim",
    priority = 1000,  -- Load early
    opts = {
      highlights = {
        Visual = { bg = "#ffff00" },
      },
    },
  }
  ```
</Accordion>
