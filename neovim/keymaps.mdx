This configuration inherits all keymaps from [LazyVim](https://www.lazyvim.org/) and can be extended with custom mappings.

## Configuration File

Keymaps are defined in `lua/config/keymaps.lua`, which is automatically loaded on the `VeryLazy` event.

```lua
-- Keymaps are automatically loaded on the VeryLazy event
-- Default keymaps that are always set: https://github.com/LazyVim/LazyVim/blob/main/lua/lazyvim/config/keymaps.lua
-- Add any additional keymaps here
```

Location: `lua/config/keymaps.lua:1-4`

<Note>
This configuration currently uses all default LazyVim keymaps without custom overrides.
</Note>

## Custom Tab Completion

The configuration modifies nvim-cmp keymaps for a more intuitive completion experience:

```lua
["<Tab>"] = cmp.mapping(function(fallback)
  if cmp.visible() then
    local entry = cmp.get_selected_entry()
    if not entry then
      cmp.select_next_item({ behavior = cmp.SelectBehavior.Select })
    else
      cmp.confirm()
    end
  else
    fallback()
  end
end, { "i", "s" }),

["<CR>"] = cmp.mapping(function(fallback)
  cmp.abort()
  fallback()
end, { "i", "s", "c" }),
```

Location: `lua/plugins/tab-complete.lua:18-35`

**Behavior:**
- **Tab**: Selects first item if completion menu is open with no selection, otherwise confirms
- **Enter**: Always inserts newline (doesn't confirm completion)
- **Ctrl-y**: Explicitly confirm completion (LazyVim default)

## Adding Custom Keymaps

To add your own keymaps, edit `lua/config/keymaps.lua`:

```lua
-- Example custom keymaps
local keymap = vim.keymap.set
local opts = { noremap = true, silent = true }

-- Quick save with Ctrl-s
keymap("n", "<C-s>", ":w<CR>", opts)

-- Clear search highlighting
keymap("n", "<leader>nh", ":nohl<CR>", { desc = "Clear search highlights" })

-- Better window navigation
keymap("n", "<C-h>", "<C-w>h", opts)
keymap("n", "<C-j>", "<C-w>j", opts)
keymap("n", "<C-k>", "<C-w>k", opts)
keymap("n", "<C-l>", "<C-w>l", opts)

-- Center search results
keymap("n", "n", "nzzzv", opts)
keymap("n", "N", "Nzzzv", opts)
```

<Note type="tip">
Use descriptive `desc` fields in your keymaps - they appear in Which-Key popup (`<leader>K`).
</Note>
